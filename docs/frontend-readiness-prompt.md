# Prompt: Upgrade Storefront for Real Users

```text
Upgrade the customer-facing frontend of the Rukhsar e-commerce website into a production-ready, real-user-ready storefront that feels premium, trustworthy, and exceptionally clean.

Context:
- Brand: Rukhsar
- Category: premium women’s ethnic and contemporary fashion
- Origin: Aurangabad, Maharashtra, India
- Brand mood: elegant, earthy-luxe, culturally rooted, refined, editorial
- Visual cues: Himroo, Paithani, Mashru, saffron, ivory, maroon, antique gold
- Current codebase already has a storefront, product pages, cart, checkout, auth, and a separate admin portal
- Do not mix admin UX into the customer-facing storefront

Primary goal:
Turn the frontend into something ready for real shoppers, not just a demo. Prioritize clarity, trust, polish, speed, readability, conversion, and mobile quality.

What to improve:

1. Visual system
- Refine typography so headings feel premium and body text feels highly readable
- Create a cleaner spacing rhythm across all pages
- Ensure buttons, cards, filters, forms, and panels feel consistent
- Reduce visual clutter and remove anything that feels prototype-like
- Keep the experience elegant and premium, not flashy or generic
- Preserve the live video hero background only if it supports readability and performance well

2. Homepage
- Make the first screen feel immediately premium and commerce-ready
- Improve hero composition, text hierarchy, CTA placement, and trust signaling
- Add polished sections for featured collections, bestsellers, heritage story, and shopping confidence
- Include subtle trust-building UI like shipping, craft quality, secure checkout, easy returns, or customer love
- Make the homepage feel designed for real conversion, not just presentation

3. Catalog and product discovery
- Improve product listing density, spacing, filter usability, and sort controls
- Make filters feel intuitive on mobile and desktop
- Improve product cards so they are clean, desirable, and scannable
- Ensure category and collection browsing feels deliberate and easy

4. Product detail page
- Make product pages feel premium and conversion-focused
- Improve image area, product info hierarchy, price presentation, size selection, care details, and calls to action
- Add trust-building blocks like shipping notes, returns clarity, fabric information, stock cues, and craftsmanship details
- Make related products feel editorial and relevant

5. Cart and checkout
- Make cart and checkout feel calm, trustworthy, and easy to complete
- Improve spacing, totals, field grouping, payment option clarity, and action hierarchy
- Optimize for mobile completion
- Remove friction and demo-like language
- Make success, loading, and error states feel polished

6. Customer account and auth
- Improve customer login, signup, account, orders, and tracking screens
- Make auth screens feel like part of a luxury brand, not default forms
- Keep login for customer users only
- Maintain separation from the admin login and admin routes

7. UX quality
- Add high-quality empty states, loading states, and error states
- Ensure text contrast, tap targets, and responsiveness are strong
- Improve visual hierarchy across all pages
- Make transitions subtle and intentional, not gimmicky
- Ensure the storefront feels stable and confidence-inspiring

8. Real-user readiness
- Audit and improve the frontend for trust, usability, and conversion
- Eliminate rough edges, inconsistent copy, awkward spacing, placeholder-feeling content, and obvious prototype UI
- Make sure the site feels like a brand customers would actually buy from

9. Technical quality
- Keep the current Next.js architecture
- Preserve the separated admin portal
- Use reusable components where appropriate
- Avoid unnecessary complexity
- Keep performance in mind, especially for video, imagery, and mobile
- Ensure production build still passes
```
